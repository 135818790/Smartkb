# RAG 知识库系统运维手册

## 第一章 系统概述

本系统是一个基于 RAG（检索增强生成）架构的企业级知识库问答系统，支持多种文档格式的导入、智能语义检索和自然语言问答。

核心组件包括：文档解析引擎、向量嵌入服务、检索引擎和大语言模型生成服务。

## 第二章 环境部署

### 2.1 硬件要求

生产环境最低配置：4 核 CPU、16GB 内存、100GB SSD 存储。推荐配置：8 核 CPU、32GB 内存、500GB SSD 存储。

GPU 可选，用于加速嵌入模型推理。推荐 NVIDIA T4 或以上，显存不低于 8GB。

### 2.2 软件依赖

- Python 3.11+
- Docker 24.0+
- PostgreSQL 15+
- Redis 7.0+

### 2.3 部署步骤

第一步：克隆代码仓库并进入项目目录。第二步：复制 .env.example 为 .env，填写必要的 API Key 和数据库连接信息。第三步：执行 docker-compose up -d 启动所有依赖服务。第四步：执行初始化脚本创建数据库表。第五步：访问 http://localhost:8000/docs 确认 API 服务正常。

## 第三章 文档管理

### 3.1 支持格式

系统支持 Markdown（.md）、纯文本（.txt）、PDF（.pdf）、Word（.docx）和网页（HTML）格式的文档导入。

PDF 文档需安装 PyMuPDF 依赖。Word 文档需安装 python-docx 依赖。

### 3.2 文档上传

通过 POST /api/documents/upload 接口上传文档。支持单文件和多文件批量上传，单个文件大小限制为 50MB。

上传后系统自动进行文档解析、文本提取、分块和向量化处理。处理进度可通过 GET /api/documents/status/{task_id} 查询。

### 3.3 文档分块策略

系统采用语义分块策略，按段落和标题层级进行分割。每个分块大小约 500 到 1000 个字符，相邻分块之间有 50 字符的重叠，确保上下文连贯。

## 第四章 问答服务

### 4.1 基本问答

通过 POST /chat 接口提交问题，系统自动检索相关文档并生成回答。响应包含答案正文和引用来源列表。

### 4.2 多轮对话

系统支持多轮对话上下文理解。通过 session_id 参数标识对话会话，系统自动将历史对话上下文注入检索和生成过程。

### 4.3 流式输出

设置 stream=true 参数启用流式输出，系统通过 Server-Sent Events 逐字返回生成内容，提升用户体验。

## 第五章 故障排查

### 5.1 常见问题

**服务启动失败**：检查端口 8000 是否被占用，执行 netstat -tlnp | grep 8000 查看。检查 .env 文件中 API Key 是否正确配置。

**文档上传后无法检索**：检查文档解析日志，确认文本提取成功。检查向量化是否完成，可通过 API 查询文档处理状态。

**回答质量差**：检查文档分块是否合理，过大的分块可能包含无关内容。尝试调整 TOP_K 参数，增加或减少检索数量。检查 Prompt 模板是否包含足够明确的指令。

### 5.2 性能优化

对包含 10000 篇以上文档的知识库，建议启用 Elasticsearch 进行关键词检索，与向量检索形成混合召回。使用 Redis 缓存热门问题的检索结果，减少重复计算。

### 5.3 日志查看

系统日志位于 logs/ 目录下。API 访问日志包含请求时间、接口路径、响应状态和耗时。检索日志包含查询文本、召回文档列表和相似度分数。
